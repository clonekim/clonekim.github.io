GTID를 이용한 Mariadb Replication

  1. 환경
  2. 마스터설정
  3. 슬레이브설정

환경

vm002: master
vm003: slave

마스터설정

GTID를 사용하기 위해 gtid_strict_mode를 활성해야함
활성화시키위해서 서버를 재시작한다

server-id = 1
character-set-server = utf8
bind-address = 0.0.0.0
log_bin = /var/lib/mysql/mariadb-bin
expire_logs_days = 2
sync_binlog = 0
slave_compressed_protocol = 1
binlog_format = row
gtid_strict_mode = 1

마스터에 접속할 슬레이브계정을 만든다

create user 'replication@vm003' identified by 'password';
grant replication slave on *.* to 'replication@vm003';
flush privileges;

슬레이브설정

[mysqld]
server-id = 2
character-set-server = utf8
bind-address = 0.0.0.0
slave_compressed_protocol = 1
binlog_format = row
read_only
gtid_strict_mode = 1

슬레이브 시작을 위해서 GTID 정보가 필요하다
마스터에서 DB에 변화를 주면 이벤트가 발생하고 아래와 같이 이벤트 번호를 확인가능하다

슬레이브에서는 마스터의 gtid를 확인을 하려면 먼저 슬레이브를 시작하여야 함

stop slave
reset slave
startslave
