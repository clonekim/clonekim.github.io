프로젝트 설정

leiningen

[org.clojure/clojurescript "1.7.228"]

plugin

[lein-cljsbuild "1.1.2"]

project.clj

(defproject hello-ring "0.1.0-SNAPSHOT"
  :description "FIXME: write description"
  :url "http://example.com/FIXME"
  :license {:name "Eclipse Public License"
            :url "http://www.eclipse.org/legal/epl-v10.html"}
  :dependencies [[org.clojure/clojure "1.7.0"]
                 [org.clojure/clojurescript "1.7.228"]
                 [ring/ring-core "1.4.0"]
                 [ring/ring-defaults "0.1.5"]
                 [ring/ring-json "0.4.0"]
                 [ring/ring-jetty-adapter "1.4.0"]
                 [compojure "1.4.0"]]
  :plugins [[lein-cljsbuild "1.1.2"]
            [lein-ring "0.9.7"]]
  :cljsbuild {:builds {:main {:source-paths ["src-cljs"]
                              :compiler {:output-to "resources/public/js/app.js"
                              	        :output-dir "resources/public/out"
                                        :optimizations :simple
                                        :pretty-print true}}}}
  :ring {:handler hello-ring.core/app}
  :main hello-ring.core)

cljs소스를 src-cljs라는 경로로 선택했다
디렉토리를 만들어야한다

~hello-ring $ mkdir src-cljs

주의

:none으로 설정 시 Google Closure Library 스크립트가 포함되지 않는다.
의존성까지 컴파일 결과에 포함하려면 :none이외의 설정으로 할것

:optimizations :none

클라이언트 측

src-cljs/hello/app.cljs

반드시 네임스페이스를 가져야한다

(ns hello.app)

(defn set-html! [el content]
  (set! (.-innerHTML el) content))

(defn main []
  (let [content "Hello world from ClojureScript"
        element (js/document.getElementById "main")]
    (set-html! element content)))

(main)

서버측

hello_ring/core.clj

(ns hello-ring.core
  (:use [ring.util.response])
  (:require [compojure.core :refer :all]
            [compojure.route :as route]
            [ring.middleware.defaults :refer [wrap-defaults site-defaults]]
            [ring.middleware.json :refer [wrap-json-response
                                          wrap-json-body
                                          wrap-json-params]]
            [hiccup.page :as page]
            [ring.adapter.jetty :refer [run-jetty]])

  (:gen-class))

(defroutes app-routes
  (GET "/" []
       (page/html5 
        [:head
         [:meta {:http-equiv "X-UA-Compatible" :content "IE=edge"}]]
        [:body
         [:div {:id "main"}
          [:button {:type "button" :id "button01"} "Click Here!"]]
         (page/include-js "js/app.js")]))
  (route/not-found "<h1>Page not found</h1>"))

(def app
  (-> app-routes
      wrap-json-body
      wrap-json-params
      wrap-json-response
      (wrap-defaults (assoc-in site-defaults [:security :anti-forgery] false))))

(defn -main []
  (run-jetty #'app {:port 8080}))

서버시작

lein ring server

다른 터미널을 열어서
src-cljs/app.cljs를 자바스크립트로 컴파일한다
한번만 컴파일한다

lein cljsbuild once

만약 소스를 주시하면서 변경 시마다 컴파일하려면

lein cljsbuild auto

이제 브라우저로 접근해보자
